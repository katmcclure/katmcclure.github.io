PROJECT 1

Client: A Vancouver-based consulting company partnered with popular Vancouver-based retailer

Overview

Project consists of multiple apps that allow customers to book personal shopping appointments with personal shoppers via video chat. I built an API that is utilized by the apps to perform CRUD operations. I also built the web app which allows customers to sign up, browse retailers, manage their personal preferences and book/cancel styling appointments with a personal shopper. Worked with an external iOS developer.

Technologies

- Built the REST API using PHP and Laravel

- Integrated middleware to protect authenticated customer and personal shopper routes from incorrect user type access

- Worked with Postman to organize, document and test all of the endpoints

- Built the web app using PHP and Laravel

- Utilized Guzzle to make calls to the API endpoints from the web app

Integrations

- Integrated with Twilio to send SMS appointment confirmations and reminders, as well as to create the video chat tokens which are used by the iOS app to begin the video chat

- Integrated with Shopify Admin API to allow creation of draft orders by personal shopper during video chat

- Integrated with Pusher to push event notifications to the iOS app

- Amazon S3 utilized to store retailer images in a central location to be accessed by all apps


PROJECT 2

Client: A Vancouver-based demolition company

Overview

A web application to track projects, documents, tasks, time entries and equipment for a demolition company. For this project, I built out the primary functionality to manage the aforementioned data through CRUD operations. This project consists of a lot of forms, data tables and reports. I also built a tool for users to create their own custom form definitions with a variety of input types and a drag and drop interface to edit field order.

I created an “offline” mode for the app, which allows workers in remote locations to view and fill out forms on their tablets in the field. The form definitions, drafts and submissions are stored in HTML local storage and submitted to the database once the user returns back online.

Technologies

- Application built using PHP and Laravel

- Used a JavaScript Service Worker to cache essential application pages and assets for viewing in offline mode

- Utilized KoolReport to render reports related to time entries by employee and project


PROJECT 3

Client: LinkBrandr

Overview

A subscription based platform for sharing links through social media while the user attaches their branding to the shared links. If the target link can be displayed in an iframe, it is displayed with a branded bar across the top of the link. If it can’t be displayed in an iframe, a splash page displays for a number of seconds with the branded information before redirecting to the provided link. I build a public website with an authenticated dashboard area for users to log in and manage their profile, branding and saved links. I also built the interface that displays the branded links and an admin site to manage users/subscriptions.

Technologies

- Application built using PHP and Laravel

- Implemented a Bootstrap theme for the public facing website

- Used Socialite to integrate Facebook authentication allowing users to sign up with their Facebook account rather than their email address.

- Utilized Goutte to scrape web urls and read response headers. When a user shares a new link, Goutte is utilized to target the site’s meta tags to grab titles and descriptions. It is also used to determine if the url has the X-Frame-Options header set, which means the website can’t be used in an iframe.

Integrations

- Stripe used for payment processing and to allow users to choose between an Annual or Monthly subscription

- Drip JavaScript API used to tag user actions for marketing purposes
